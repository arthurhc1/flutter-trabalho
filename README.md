# flutter-server
backend Java Spring que persiste os dados via API REST enviadas pelo app
para rodar deve-se executar o arquivo "FlutterServerApplication.java", o projeto executará localmente na porta 8080.

# flutter_trabalho
app de tarefas, onde pode-se criar uma tarefa, marca-la ou desmarca-la e também excluí-la, feita em flutter, utilizando o retrofit para o consumo da api do backend, possui também troca de temas.
para rodar é só executá-lo normalmente em um emulador(não fiz configuração para troca do Ip do servidor no app), o app consumirá o server pelo ip "10.0.2.2" na porta 
"8080".
